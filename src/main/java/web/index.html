实验简介

本项目是《HTTP 作业 / Socket 实验》的演示实现，完全基于 Java 原生 Socket API， 搭建了一个简易的 HTTP 服务器与 HTTP 客户端（命令行 + GUI），用于理解 HTTP 报文、状态码和长连接等核心概念。

① 基础要求说明

不允许基于 Netty 等第三方网络框架，必须完全使用 Java 原生 Socket API 实现 HTTP 客户端和服务器端程序。

JavaSocket原生实现

② IO 模型不限

实现不限制使用的 IO 模型，可以是传统 BIO、NIO 或 AIO， 本项目示例采用较为直观的 BIO 方式，便于理解请求解析与响应发送流程。

BIONIOAIO

③ HTTP 客户端功能

客户端能够构造并发送完整的 HTTP 请求报文，展示服务器返回的响应头和响应体， 同时提供命令行模式和 GUI 图形界面两种交互方式。

命令行GUIHTTP 客户端

④ 重定向 / 缓存状态码处理

客户端对 301、302、304 等状态码做了相应处理： 能够跟随 301/302 重定向跳转到新地址，并识别 304 Not Modified 与缓存相关的响应。

301302304

⑤ 服务器端 HTTP 支持

服务器端支持 GET 与 POST 请求，能够返回常见状态码： 200、301、302、304、404、405、500，并通过 Connection 头实现 HTTP/1.1 长连接。

GETPOST长连接

⑥ MIME 与静态资源

服务器端实现了基础 MIME 类型映射，至少支持三种类型， 包含一种非文本类型（如图片），用于正确返回 HTML、文本和二进制文件等静态资源。

MIME静态文件图片

⑦ 注册 / 登录接口（实验扩展功能）

在以上 HTTP 能力的基础上，服务器端额外实现了简单的注册与登录接口： 用户信息可以保存在内存或文件中，用于验证用户名与密码。 可以通过 Postman、命令行客户端或 GUI 客户端等方式发送 POST 请求进行测试。

实现目标是理解 HTTP 与 Socket 在真实业务场景中的结合方式，而非构建完整的生产级用户系统。
